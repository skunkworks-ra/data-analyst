# 05 — Execution Workflow

How to generate, execute, and validate simulation scripts.

## Script structure

Generate a single self-contained Python script. Use this template skeleton:

```python
#!/usr/bin/env python3
"""
MS Simulation: {description}
Generated by ms-simulator skill
"""
import os
import shutil
import numpy as np
from casatools import simulator, table, componentlist, measures, quanta
# Only import image, coordsys if using image-based sky models
# Only import casatasks if using ft/tclean for prediction

sm = simulator()
tb = table()
cl = componentlist()
me = measures()
qa = quanta()

msname = "{output_path}"

# Clean up any previous run
if os.path.exists(msname):
    shutil.rmtree(msname)

# ── Stage 1: MS frame ──────────────────────────────────────
{antenna_config_code}

sm.open(ms=msname)

# Observatory location
pos = me.observatory("{telescope}")
sm.setconfig(
    telescopename="{telescope}",
    x=x, y=y, z=z,
    dishdiameter=diam,
    mount=["ALT-AZ"] * len(x),
    antname=antnames,
    coordsystem="{coordsys}",
    referencelocation=pos,
)

sm.setfeed(mode="perfect R L", pol=[""])
{spw_setup_code}
{field_setup_code}

sm.setlimits(shadowlimit=0.001, elevationlimit="8deg")
sm.setauto(autocorrwt=0.0)
sm.settimes(integrationtime="{inttime}s", usehourangle=True,
            referencetime=me.epoch("UTC", "{refdate}"))

{observe_code}

sm.close()

# ── Stage 2: Sky model ─────────────────────────────────────
{sky_model_code}

# ── Stage 3: Predict visibilities ───────────────────────────
sm.open(ms=msname)
sm.predict({predict_args})
sm.close()

# Copy MODEL_DATA to DATA
tb.open(msname, nomodify=False)
tb.putcol("DATA", tb.getcol("MODEL_DATA"))
tb.close()

# ── Stage 4-5: Noise and corruption ────────────────────────
{corruption_code}

print(f"Simulation complete: {{msname}}")
```

## Antenna config loading

**For known telescopes** (VLA, MeerKAT, uGMRT): locate the shipped config file.

```python
import casatools
datapath = os.path.join(casatools.casadata.datapath, "alma", "simmos")
cfg_file = os.path.join(datapath, "vla.b.cfg")

x, y, z, diam, antnames = [], [], [], [], []
with open(cfg_file) as f:
    for line in f:
        line = line.strip()
        if line and not line.startswith("#"):
            parts = line.split()
            x.append(float(parts[0]))
            y.append(float(parts[1]))
            z.append(float(parts[2]))
            diam.append(float(parts[3]))
            antnames.append(parts[4] if len(parts) > 4 else f"A{len(x)-1:02d}")
x, y, z, diam = np.array(x), np.array(y), np.array(z), np.array(diam)
```

**For custom arrays**: generate positions inline (see 02-antenna-configs.md).

## Field setup

```python
sm.setfield(sourcename="3C286",
            sourcedirection=me.direction("J2000", "13h31m08.3s", "+30d30m33s"))
```

Multiple fields: call `sm.setfield` once per source.

## Observe schedule

```python
# Single source, single SPW
sm.observe("3C286", "spw0", starttime="-0.5h", stoptime="+0.5h")

# Multi-source schedule
sm.observe("3C286", "spw0", starttime="-2.5h", stoptime="-2.42h")  # flux cal
sm.observe("target", "spw0", starttime="-2.4h", stoptime="-2.0h")  # target
sm.observe("phasecal", "spw0", starttime="-2.0h", stoptime="-1.97h")  # phase cal
# ... repeat pattern ...
```

## Validation

After execution, validate the output MS exists and is well-formed:

1. **Quick check**: `ls -la {output_path}` — confirm directory exists
2. **listobs**: Run `casatasks.listobs(vis=msname)` for a summary
3. **ms_inspect tools**: Use `ms_observation_info`, `ms_field_list`,
   `ms_antenna_list` to confirm structure matches simulation parameters

## Common pitfalls

### sm.predict fails with "no model"
- Component list path must be absolute
- If using an image model, the image must exist on disk before `sm.predict`

### Empty DATA column
- `sm.predict` writes to MODEL_DATA, not DATA
- You must copy MODEL_DATA → DATA with `tb.putcol` before noise injection
- `sm.corrupt` (noise) adds to DATA, not MODEL_DATA

### MS already exists
- `sm.open` will fail if the MS path already exists
- Always `shutil.rmtree` before creating a new MS

### Coordinate system mismatch
- Shipped VLA/MeerKAT configs use `global` (ITRF/XYZ) coordinates
- Custom arrays often use `local` (East-North-Up) coordinates
- The `coordsystem` parameter in `sm.setconfig` must match

### Missing observatory in CASA database
- `me.observatory("VLA")` works, `me.observatory("MyArray")` does not
- For custom telescopes, create a position manually:
  `me.position("ITRF", "q1", "q2", "q3")` or use a known observatory
  as the reference and note it in the output

### Hour angle range
- `usehourangle=True` means starttime/stoptime are relative to transit
- `-0.5h` to `+0.5h` = 1 hour centered on meridian transit
- For full tracks, use `-6h` to `+6h` (12-hour synthesis)

### Large MS warning
Approximate MS size: `N_bl * N_chan * N_pol * N_time * 16 bytes`
where `N_bl = N_ant * (N_ant - 1) / 2` and `N_time = duration / inttime`.
Warn if estimated size exceeds 1 GB.
